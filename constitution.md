# 共通憲法（Global Constitution）

すべてのプロジェクトで遵守すべきルール。エージェントはタスク実行前にこの憲法を必ず読み込むこと。

## コーディング規約

- Python は 3.11 を使用する（3.14 は langfuse・langchain 等と互換性問題があるため禁止）
- 仮想環境は `.venv` を使用し、グローバル環境への pip install は禁止
- ファイルの文字コードは UTF-8 統一
- インデントは半角スペース4つ
- 関数・クラスには必ず docstring を記述する
- import 文は標準ライブラリ → サードパーティ → ローカルの順にグループ化する

## モデル選択・セキュリティ

- API キー・トークンは `.env` に格納し、コードにハードコードしない
- `.env` と `mcp_config.json` は `.gitignore` に含め、GitHub に push しない
- `confidential: true` のプロジェクトではクラウド API（OpenAI・Anthropic・Google等）を使用禁止。ローカルモデル（Ollama）のみ使用を強制する
- `confidential: false` のプロジェクトでは、タスクの複雑度に応じてローカルモデルとクラウド API を自動切替してよい
- モデル選択の優先順位：
  1. `projects.json` の `model_override`（プロジェクト指定があれば最優先）
  2. タスク複雑度スコアによる自動選択（高複雑度 → クラウド API、低複雑度 → ローカル）
  3. `projects.json` の `defaults.model`（フォールバック）
- クラウド API の利用時はコストを記録し、Langfuse でトレースすること

## ファイル操作

- ファイルの作成・編集は必ず指定されたプロジェクトディレクトリ内で行う
- `~/projects/` 以外のディレクトリへの書き込みは禁止
- 既存ファイルを上書きする前に、変更内容をレビューで確認する

## エラー対応

- エラーが発生した場合、まず原因を特定してから修正する
- 同じエラーが過去に発生していないか `lessons.json` を確認する
- 修正後は必ず構文チェック（py_compile）を実行する
- 3回リトライしても解決しない場合は、人間に報告して判断を仰ぐ

## テスト

- ファイルを作成したら構文チェックを必ず実行する
- テストコードがある場合は pytest を実行して結果を報告する
- テスト失敗時はエラー内容を分析してから修正を行う

## コミュニケーション

- 結果報告は日本語で行う
- コードブロック内のコメントは日本語可（ただし変数名・関数名は英語）
- エラー報告には「何が起きたか」「原因は何か」「どう修正したか」を含める

## 禁止事項

- `rm -rf /` 等の破壊的コマンドの実行
- 無限ループを含むコードの実行（timeout 120秒で強制停止）
- 本番環境への直接デプロイ（必ずレビューを経ること）
- パッケージの無断アップグレード（バージョン互換性を事前確認すること）

# プロジェクト固有憲法テンプレート

各プロジェクトのルートに `constitution.md` を配置する場合、以下をベースにカスタマイズすること。

テンプレート配置先の例：`~/projects/{project_name}/constitution.md`
```markdown
# {project_name} 憲法

## プロジェクト固有ルール
- （このプロジェクト特有のコーディング規約を記載）

## 使用ライブラリ制約
- （使用可・使用不可のライブラリを記載）

## デプロイ・環境
- （開発環境・ステージング・本番の区別を記載）

## 注意事項
- （過去に問題が起きたポイントなど）
```
